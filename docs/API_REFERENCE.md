# API Reference

Base URL: `http://localhost:8080`

The machine-readable OpenAPI 3.0 spec is generated by the gateway:

```bash
cd gateway && go run . --openapi-spec > openapi.json
# or: make openapi
```

## Authentication

- **`none`** (default): the gateway only accepts requests from `127.0.0.1`/`::1`.
- **`token`**: send `Authorization: Bearer <token>`. Tokens are scoped per tool
  (`search`, `files`, `exec`, `browser`) — see `config.yaml`.

All error responses use the shape `{ "error": "<message>" }`.

## System

### `GET /health`

Liveness + per-tool readiness. No auth required.

```json
{ "status": "ok", "version": "0.1.0", "tools": { "search": "ready" } }
```

### `GET /tools`

List registered tools and their status.

## Search

### `POST /search`

| Field     | Type       | Required | Description                          |
| --------- | ---------- | -------- | ------------------------------------ |
| `query`   | string     | yes      | Search query                         |
| `engines` | string[]   | no       | Restrict to specific engines         |
| `page`    | integer    | no       | 1-based page number (default 1)      |
| `lang`    | string     | no       | Language filter (e.g. `en`)          |

**Response**

```json
{
  "query": "golang",
  "page": 1,
  "count": 2,
  "results": [
    { "title": "...", "url": "...", "snippet": "...", "engine": "google" }
  ]
}
```

## Files

All file paths are relative to the sandbox root (`files.sandbox_root`).

| Endpoint            | Body                                    | Description            |
| ------------------- | --------------------------------------- | ---------------------- |
| `POST /files/read`  | `{ "path": "a.txt" }`                   | Read file contents     |
| `POST /files/write` | `{ "path": "a.txt", "content": "..." }` | Write/overwrite a file |
| `POST /files/list`  | `{ "path": "." }`                       | List a directory       |
| `POST /files/delete`| `{ "path": "a.txt" }`                   | Delete a file          |
| `POST /files/move`  | `{ "path": "a.txt", "destination": "b.txt" }` | Move/rename    |

## Exec

### `POST /exec` — synchronous execution

| Field      | Type    | Required | Description                       |
| ---------- | ------- | -------- | --------------------------------- |
| `code`     | string  | yes      | Source code                       |
| `language` | string  | yes      | e.g. `python`, `node`, `bash`, `c`, `cpp`, `rust`, `java`, `dotnet` |
| `stdin`    | string  | no       | Standard input                    |
| `timeout`  | integer | no       | Seconds (bounded by `exec.max_timeout`) |

**Response**

```json
{ "stdout": "Hello\n", "stderr": "", "exit_code": 0, "duration": 0.12 }
```

### `POST /exec/async` — enqueue a job → `202`

```json
{ "job_id": "job_...", "execution_id": "...", "status": "queued", "position": 1, "language": "python" }
```

### `GET /exec/{id}` — poll job status

### `DELETE /exec/{id}` — cancel a queued/running job

## Browser

| Endpoint                       | Description                     |
| ------------------------------ | ------------------------------- |
| `POST /browser/session`        | Create a session (`{ "browserType": "chromium" }`) |
| `GET /browser/session/{id}`    | Session details                 |
| `DELETE /browser/session/{id}` | Close a session                 |
| `POST /browser/action`         | Run an action in a session      |

Action body:

```json
{
  "session_id": "<id>",
  "action": { "type": "navigate", "url": "https://example.com" }
}
```

Supported action types include `navigate`, `click`, `type`, `screenshot`, and
`pdf`. See [BROWSER_API.md](BROWSER_API.md) for the complete list.
